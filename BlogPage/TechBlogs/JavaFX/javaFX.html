Style Choice

Color

I don't think my color usage is something that I would put into production on real products I develop. I do like color for my personal projects because it helps visual distinguish different focus points on screen. Using the muted colors also prevents the color from becoming distracting and drawing attention away from what the user is actually trying to accomplish on screen. My color choice is an attempt to create a forest theme.

Button Placement

I placed the exit button in red on the bottom right of the program for a specific reason. I recently completed a UX class and had to submit a website I developed for review by peers. No one that reviewed my website ever found the question mark icon at the bottom right of the page that would have led them to the FAQ page. I did color the question mark blue, so I want to determine if it was color or button location that led people to not see it. Other than wanting to experiment with this idea, I don't particularly like the location of the exit button.

I wanted the add, modify, and delete buttons to resemble the old game boy start and select buttons. I think I got the shape close but the color of the original buttons are too dark to work with black text so I chose to just utilize the shape.

What next

Next I want to work on getting the add buttons to successfully open a new window and display a new styled page for adding parts and adding products. This will be my first time working with Event handlers and opening new windows.

12/18/2022

Finished

I finished the project about a week ago. I had some bugs to work out in validation and that caused the final submission of the project back longer than I had intended. However, it was a good experience in debugging and gave me perspective on the level of diligence required to anticipate potential problems and implement solutions to mitigate eliminate. I also ended up formatting the columns on the table to display product and part price columns in currency format.

This caused me a problem when using the search function for the parts and products. When a search would return a null result it would throw an exception and break the runtime. Originally I had the code for formatting the price columns in the initialize method for the main screen controller. However after encountering the search bug I determined it would be better to create a method to handle the column formatting. In the method I placed a conditional that would return if the result set for the search was empty. This prevents the column formatter from attempting to format an empty table.

Add/Modify Part Form

The add and modify part forms were the easiest for me to handle. Text fields to collect user input on parts with radio buttons to choose which type of part that is being modified or created. An id is automatically generated when the user clicks the save button.
